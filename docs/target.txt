Цель (Target).

Цели составляют основу дерева сборки (Tree) - только они могут являться его узлами.
Цель имеет тип и название. Название может быть полное (Label) и краткое (Name).
Полное название уникально для каждой цели во всём проекте.

Полное название выглядит так:

  //path/to/folder:name

где name - это краткое название.
Подразумевается, что цели привязаны к пути (конкретно к папке) в файловой системе.
Корнем всех путей является корневая папка проекта.

Цели бывают таких типов:
  - группа (group)
  - программа (executable)
  - статическая библиотека (static_library)
  - динамическая библиотека (shared_library)
  - компонент (component)
  - набор исходников (source_set)
  - действие (action)

Цели могут зависеть друг от друга, но только отношение зависимостей строго задано:

  - группа может зависеть от:
    * группы
    * программы
    * статической библиотеки
    * динамической библиотеки
    * компоненты
    * действия

  - программа может зависеть от:
    * группы
    * статической библиотеки
    * динамической библиотеки
    * компоненты
    * набора исходников
    * действия

  - статическая библиотека может зависеть от:
    * группы
    * статической библиотеки
    * компоненты
    * набора исходников
    * действия
